# Screenshot Capture Pipeline

This page documents how the automated screenshots in this site are produced. All images are generated by a headless Playwright script against a live dev server — no manual cropping or editing.

---

## Prerequisites

| Requirement | Version | Check |
|---|---|---|
| Node.js | 18 or higher | `node --version` |
| npm | 8 or higher | `npm --version` |
| DeepPhe Visualizer running | port 3000 | `curl -s http://localhost:3000 | grep -c DeepPhe` |
| DeepPhe Data API running | port 3333 | `curl -s http://localhost:3333/health` |
| Playwright browsers installed | — | see below |

### Install Playwright browsers (first time only)

`playwright` is already listed as a project dependency. After `npm install`, install the Chromium browser binary:

```bash
npx playwright install chromium
```

You only need to do this once per machine (or after a Playwright version bump).

---

## Running the capture script

### Step 1 — Start both servers

In one terminal, start the frontend dev server:

```bash
npm start
# Waits until http://localhost:3000 is serving the app
```

In a second terminal, start the DeepPhe Data API (path will vary):

```bash
cd /path/to/dphe-data-api
PORT=3333 npm start
```

### Step 2 — Run the capture script

```bash
npm run capture:screenshots
```

This runs `scripts/capture-screenshots.mjs` headlessly. It opens a 2200×1400 Chromium window, navigates through all documented routes, and writes PNGs to `output/playwright/`.

**Custom app URL** (if your dev server is on a different port):

```bash
APP_URL=http://localhost:3001 npm run capture:screenshots
```

### Step 3 — Check the output

```
output/playwright/
  01-home.png
  02-filters-overview.png
  ...
  33-patient-summary-card.png
  capture-summary.json          ← per-screenshot status (captured / fallback)
```

Open `capture-summary.json` to see which screenshots used fallback paths and why. `missingStates` lists anything that could not be captured at runtime (usually because the backend did not return data for that UI state).

## What the script captures

| Range | Subject |
|---|---|
| 01 | Home route |
| 02–03 | Filters overview and Identified Patients panel |
| 04–07 | Theme selector (open state, Obsidian, Solstice, Vapor) |
| 08–20 | All filter set sections and individual filter cards |
| 21 | Active filter selection state |
| 22–25 | Patient details grid (overview, column menu, expanded row, empty search) |
| 32–33 | Embedded patient view drawer and Patient Summary Card |
| 30–31 | Debug view and Accessibility Statement |

### Data-dependent screenshots

Captures **32** and **33** require the backend to return patient data for at least one patient in the current cohort. If the API is unavailable or returns an empty cohort, those screenshots fall back to a 1×1 placeholder and the reason is logged in `capture-summary.json`.

## Building the full docs site

`npm run docs:build` copies screenshots into `docs/assets/screenshots/`, then builds the MkDocs site to `site/`:

```bash
npm run docs:build
```

The first run creates a Python virtualenv at `.venv-mkdocs/` and installs `mkdocs-material` automatically. Subsequent runs reuse it.

Open `site/index.html` in a browser to review the built site locally.

### One-command review pipeline

```bash
npm run docs:review
```

This chains capture → build → PDF export in sequence. Use this before sharing a review package.

## Troubleshooting

**Script exits immediately with "browser not found"**
Run `npx playwright install chromium` and retry.

**All screenshots are 1×1 placeholders**
The app server is not responding. Confirm `http://localhost:3000` loads in a browser before running the script.

**Captures 32–33 are fallbacks**
The Data API returned no patients for the current cohort, or the "Show in Document Viewer" button was not reachable. Check that the backend is running and returning data at `/deepphe/cancers/instances/patients`.

**`docs:build` fails with "Missing screenshot: …"**
Run `npm run capture:screenshots` first, or check `capture-summary.json` to see which file was not written.

**MkDocs build fails with Python errors**
Delete `.venv-mkdocs/` and re-run `npm run docs:build` to recreate the virtualenv.
